Introduction to WSML

The Web Service Modeling Language (WSML) provides a formal syntax and semantics for the Web Service Modeling Ontology (WSMO). WSML is a language for semantic web services based on logical formalisms.

As a family of languages, WSML consists of several variants that provide different levels of logical expressiveness. This flexibility allows users to make different trade-offs between expressivity and tractability according to their needs.

WSML enables the formal description of:

  • Ontologies that provide the terminology used by other components
  • Web services including their functional and non-functional properties
  • Goals that represent user desires and requests
  • Mediators for handling interoperability issues

WSML-Core

WSML-DL

An extension of WSML-Core that provides the expressive power of Description Logics, corresponding to the SHIQ(D) Description Logic.

WSML-Flight

An extension of WSML-Core based on a logic programming variant of F-Logic, including features such as meta-modeling and constraints.

WSML-Rule

Extends WSML-Flight to a full-fledged logic programming language, allowing function symbols and unsafe rules.

WSML-Full

Unifies all WSML variants under a First-Order Logic with non-monotonic extensions, providing the most expressive capabilities.

WSML Syntax

WSML has both a normative human-readable syntax and XML and RDF serializations for machine processing. The syntax is designed to be intuitive for users familiar with the frame-based style of modeling.

WSML conceptual syntax defines how to specify ontologies, goals, web services and mediators:

namespace {_"http://example.org/wsml/ontologies/example#", dc _"http://purl.org/dc/elements/1.1#"} ontology _"http://example.org/wsml/ontologies/example" nonFunctionalProperties dc#title hasValue "Example Ontology" dc#creator hasValue "Jane Doe" endNonFunctionalProperties concept Person hasName ofType _string hasAge ofType _integer hasChild ofType Person instance JohnDoe memberOf Person hasName hasValue "John Doe" hasAge hasValue 35

WSML logical expressions are used to define constraints, axioms, and capabilities:

axiom AgeRestriction definedBy ?x[hasAge hasValue ?age] and ?age < 18 implies ?x[memberOf Child]. capability BookingCapability precondition definedBy ?user[hasAccount hasValue ?account] and ?account[accountBalance hasValue ?balance] and ?balance > 0. effect definedBy ?user[hasReservation hasValue ?reservation].

WSML can be serialized in XML for machine processing:

<?xml version="1.0" encoding="UTF-8"?> <wsml:wsml xmlns:wsml="http://www.wsmo.org/wsml/wsml-syntax#" xmlns:xsd="http://www.w3.org/2001/XMLSchema#"> <wsml:ontology name="http://example.org/wsml/ontologies/example"> <wsml:nonFunctionalProperties> <wsml:property name="http://purl.org/dc/elements/1.1#title"> <wsml:value type="xsd:string">Example Ontology</wsml:value> </wsml:property> </wsml:nonFunctionalProperties> <wsml:concept name="Person"> <wsml:attribute name="hasName" type="ofType" dataType="xsd:string"/> <wsml:attribute name="hasAge" type="ofType" dataType="xsd:integer"/> <wsml:attribute name="hasChild" type="ofType" range="Person"/> </wsml:concept> </wsml:ontology> </wsml:wsml>

WSML Examples

Below are examples of WSML in different contexts:

A simple e-commerce ontology described in WSML:

namespace {_"http://example.org/ecommerce#", dc _"http://purl.org/dc/elements/1.1#"} ontology _"http://example.org/ecommerce" nonFunctionalProperties dc#title hasValue "E-Commerce Ontology" dc#description hasValue "Basic concepts for e-commerce applications" endNonFunctionalProperties concept Product hasName ofType _string hasPrice ofType _decimal hasCategory ofType Category inStock ofType _boolean concept Category hasName ofType _string hasParentCategory ofType Category concept Customer hasName ofType _string hasEmail ofType _string hasAddress ofType Address concept Order hasCustomer ofType Customer hasOrderDate ofType _dateTime hasItems ofType OrderItem hasTotalPrice ofType _decimal concept OrderItem hasProduct ofType Product hasQuantity ofType _integer

A web service for product ordering:

namespace {_"http://example.org/services/ordering#", dc _"http://purl.org/dc/elements/1.1#", ec _"http://example.org/ecommerce#"} webService _"http://example.org/services/ProductOrderingService" nonFunctionalProperties dc#title hasValue "Product Ordering Service" dc#description hasValue "Service for ordering products" endNonFunctionalProperties capability OrderingCapability nonFunctionalProperties dc#description hasValue "Capability of ordering products" endNonFunctionalProperties precondition definedBy ?customer[memberOf ec#Customer] and ?product[memberOf ec#Product] and ?product[ec#inStock hasValue true] and ?quantity[memberOf _integer] and ?quantity > 0. effect definedBy ?order[memberOf ec#Order] and ?order[ec#hasCustomer hasValue ?customer] and ?orderItem[memberOf ec#OrderItem] and ?orderItem[ec#hasProduct hasValue ?product] and ?orderItem[ec#hasQuantity hasValue ?quantity] and ?order[ec#hasItems hasValue ?orderItem]. interface OrderingInterface choreography OrderingChoreography // Choreography definition here orchestration OrderingOrchestration // Orchestration definition here

A goal to find and order a product:

namespace {_"http://example.org/goals/ordering#", dc _"http://purl.org/dc/elements/1.1#", ec _"http://example.org/ecommerce#"} goal _"http://example.org/goals/OrderProduct" nonFunctionalProperties dc#title hasValue "Order Product Goal" dc#description hasValue "Goal of ordering a specific product" endNonFunctionalProperties capability postcondition definedBy ?customer[memberOf ec#Customer] and ?customer[ec#hasName hasValue "John Smith"] and ?product[memberOf ec#Product] and ?product[ec#hasName hasValue "Smartphone XYZ"] and ?order[memberOf ec#Order] and ?order[ec#hasCustomer hasValue ?customer] and ?orderItem[memberOf ec#OrderItem] and ?orderItem[ec#hasProduct hasValue ?product] and ?orderItem[ec#hasQuantity hasValue 1] and ?order[ec#hasItems hasValue ?orderItem].

History & Evolution

Early 2000s

Research on semantic web services begins with frameworks like OWL-S and WSMF (Web Service Modeling Framework).

2004

WSMO (Web Service Modeling Ontology) is developed as a conceptual model for semantic web services.

2005

WSML is introduced as a family of languages to formalize WSMO, providing different levels of logical expressiveness.

2006-2008

Refinement of WSML variants, with implementations in various reasoning systems and toolsets like WSMT (Web Service Modeling Toolkit).

2010s

Integration with other semantic web technologies and standards, continuing development within the broader semantic web services ecosystem.

Resources & Tools

Several tools and frameworks have been developed for working with WSML:

  • WSMO4J - A Java API for programmatic creation, manipulation and access to WSMO elements
  • WSML2Reasoner - A framework for reasoning with WSML ontologies
  • WSMT (Web Service Modeling Toolkit) - An integrated environment for semantic web services
  • WSML Validator - For checking the syntax and semantics of WSML documents
  • WSML Validator - For checking the syntax and semantics of WSML documents
  • WSMO Studio - An Eclipse-based integrated development environment for WSMO
  • WSMX (Web Service Execution Environment) - A reference implementation of WSMO
